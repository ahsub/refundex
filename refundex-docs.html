Refundex v111

Refundex ist ein browserbasiertes Steuerhilfe-Tool für IBKR/CapTrader-Kunden. Alle Berechnungen laufen lokal auf Ihrem Gerät — keine Daten verlassen Ihren Browser.

Datenschutz: Refundex verarbeitet alle Steuerdaten lokal im Browser. Es werden keine Finanzdaten an externe Server übertragen.

Voraussetzungen

  • CapTrader oder IBKR Konto (Kontoumsatz-Flex-Query Berechtigung)
  • Moderner Browser (Chrome, Firefox, Safari, Edge)
  • Keine Installation erforderlich

Schnellstart

1
Flex Query in CapTrader erstellen — einmalige Einrichtung, danach wiederverwendbar. Anleitung →
2
CSV exportieren — für jedes Steuerjahr eine Datei. Alle Jahre können gleichzeitig hochgeladen werden.
3
KAP-Assistent öffnen und CSV(s) in die Upload-Zone ziehen.
4
Steuerjahr wählen und Ergebnisse prüfen. Bei Gemeinschaftskonto: Kontoart auf "50/50" stellen.
5
DOCX herunterladen — fertige Begleitdokumentation für das Finanzamt.

Was ist eine Flex Query?

Eine Flex Query ist ein konfigurierbarer Datenexport von IBKR/CapTrader. Im Gegensatz zum Standard-Jahresauszug enthält sie exakte IBKR-Tageskurse und den bereits berechneten FIFO-Gewinn/Verlust je Trade.

Flex Query einrichten

Einmalige Einrichtung — danach können Sie die Query für jedes Steuerjahr neu ausführen.

Schritt-für-Schritt (CapTrader Web)

1
Login auf clientam.com → Performance & Berichte → Flex Queries
2
Kontoumsatz-Flex-Query → + (Neu erstellen) oder bestehende Query bearbeiten
3
KI-Assistent verwenden — Prompt eingeben (siehe unten) oder Felder manuell konfigurieren
4
Format: CSV · Spaltenüberschriften: Ja · Abschnittscodes: Nein
5
Zeitraum: Benutzerdefiniert — z.B. 01.01.2024 bis 31.12.2024
6
Speichern & Ausführen — CSV herunterladen

KI-Prompt für Flex Query

Kopieren Sie diesen Prompt in den KI-Assistenten von CapTrader:

✦ Empfohlener Prompt
Flex Query: Account Information (Name, AccountType) + Trades (FifoPnlRealized, FXRateToBase, CostBasis, OpenCloseIndicator, AssetClass, ISIN, Multiplier, Put/Call, Strike, Expiry) + Cash Transactions (Dividenden, WHT, Zinsen). CSV.
197 Zeichen
Account Information ermöglicht die automatische Erkennung von Einzel- vs. Gemeinschaftskonten und der Kontoinhaber-Namen direkt aus der CSV.

Gemeinschaftskonto

Bei gemeinsamen Depots (Ehegatten, GbR) teilt Refundex alle Beträge automatisch hälftig auf.

1
Kontoart auf "Gemeinschaftskonto 50/50" stellen
2
Ehegatte auswählen (Inhaber 1 oder Inhaber 2)
3
Refundex erstellt beim Download zwei separate DOCX-Dokumente — eines pro Steuerpflichtigen
Nach dem Update des Flex Query Prompts um "Account Information" erkennt Refundex Gemeinschaftskonten automatisch am Schlüsselwort AND im Kontonamen (z.B. "AXEL HILDEBRAND AND CHRISTA F HILDEBRAND").

ko-fifo.js Beta

FIFO-Engine nach §20 Abs.4 Satz 7 EStG. Berechnet Verlustverrechnungstöpfe aus Trade-Historien.

Hinweis: Bei Verwendung der Flex Query mit FifoPnlRealized wird ko-fifo.js nicht mehr benötigt — IBKR liefert den FIFO-G/V direkt.

ko-fx.js

Lädt EZB-Tageskurse via Frankfurter.app API (CORS-Proxy). Wird als Fallback verwendet wenn kein FXRateToBase in der Flex Query vorhanden ist.

ko-dba.js

Datenbank der Doppelbesteuerungsabkommen für 18 Länder. Berechnet den maximal anrechenbaren Quellensteuersatz je Land für die Anlage KAP Z.41.

Haftungsausschluss

Refundex ist ein Hilfsmittel zur Vorbereitung der Steuererklärung, kein steuerrechtliches Beratungsinstrument. Die berechneten Werte sind eine Vorabschätzung auf Basis der IBKR-Daten. Für die korrekte steuerliche Behandlung ist ein Steuerberater hinzuzuziehen, insbesondere bei:
  • Call/Put Assignments und deren Behandlung als Z.8 oder Z.12
  • Vorabpauschale auf ETFs (§18 InvStG)
  • Gold-ETCs (§23 EStG)
  • Verlustvorträgen aus Vorjahren
